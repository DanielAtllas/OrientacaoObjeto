Introdução - Analytics

É muito comum escutarmos sobre o tema de programação orientada a objetos, que por sua vez é um "nível" a mais sobre a pogramação com novas capacidades e estratégias para se compilar. Diante disso, temos uma linguagem que se enciiaxa muito bem para o estudo deste conteúdo, a linguagem java. Porém, saiba que orientação a objetos não fica sujeita apenas ao Java, mas a todas as formas de linguagem. A questão, é que o java se capacita de maneira mais eficiente e fácil ao aprendizado deste conteúdo.

Mas o que viria a ser essa chamada orientação a objetos? E o que à difere de uma programação estruturada?

Em termos gerais, a definição/explcação para a orientação a objetos é uma programação baseada na composição e interação entre divrsas unidades de softwares chamadas de objetos, ou seja, é a divisão de um software (ou script/projeto) em pequenas partes, de forma que ao juntar essas partes, sua composição estará completa. Por mais que pareça confuso, é na verdade muito simples. Imagine um script de mais de 500 linhas de comandos, entre eles variáveis, funções e etc, escrita em C. Para realizar a manutenção ou até mesmo a escrita deste script, uma atenção muito clínica deve ser aplicada, pois o programador em questão, terá de analisar linha por linha de todo o script, logo este é um trabalho exaustivo e de muita atenção.

A orientação a objetos vem para facilitar esta operação com scripts/projetos, uma vez que um script de 500 linhas, como citado acima, poderá ser dividido em 4 scripts de 250 linhas, os quais ao serem mesclados irão gerar um mesmo resultado. Percebeu o quanto é facilitador? Pois dessa forma, ao invés de termos 1 programador trabalhando em um script de 500 linhas (o que é exaustivo, clínico e arriscado), podemos ter 4 programadores focados em apenas partes deste script, dividindo assim o problema para torna-lo mais fácil. Este é o objetivo da orinetação a objetos.

    Perceba então que algumas das demais linguagens de programação, como no caso da linguagem C, necessitam particulamente de diferentes scripts para a execução em determinado sistema operacional, ou seja, para ter um mesmo programa que execute nos 3 principais S.O.'s do mercado, 3 diferentes scripts devemser escritos (por mias que possuam a mesma lógica e objetivo, mas com uma diferente abordagem). O que no caso do Java não se faz necessário, pois um único script escrito nesta linguagem possui uma capacidade de rodagem em qualquer ambiente, sem preocupação.

    Essa capacidade se faz possível pela utilização da JVM que é aclopada e trabalhada junto ao pacote Java. Essa JVM (ou Java Vitual Machine) funciona como um compilador universal, que ao pegar um script java, este será levado até a VM do java, que irá analisar e aplica-lo no determinado sistema operacional, retornando ao usuário o resultado de maneia rápida. Diante disso, podemos dizer que a JVM serve como ponto de compilador/tradutor da linguagem para os determinados sistemas operacionais e ambientes.

  • Siglas usadas pelo Java

    • Assim como a JVM citada acma, muitas outras siglas são usadas no cotidiano que envolve o java, as quais podem gerar uma certa confusão na cabeça das pessoas, principalmente nos que estão ingressando neste universo a primeira viagem. Portanto, saiba que obiviamente essas siglas possuem um segnificado que indica algum tipo deserviço ou plataforma gerenciada e administrada pelo Java, onde as 3 principais são:

    • JVM - Ou Java Virtual Machine, que vem a ser uma máquina virtual Java, funcionando como um compilador de scripts, o qual vem acompanhado/imbutido a IDE de escrita de códigos.

    • JRE - Ou Java Runtime Environment, em a ser o ambiente de execução Java, ou seja, o ambiente que fará o prcesso de execução do que foi compilado pela JVM. Logo, temos que uma JRE é composta pela própria JVM e as bibliotecas usadas pela linguagem Java, contendo as camadas necessárias para execução do script.

    • JDK - Ou Java Development Kit, este sim pode ser tido como o pacote inteiro do Java, ou melhor, o pacote inicial para desenvolvimento Java, o qual é composto pela JRE e o ambiente de escrita para o código Java, junto as ferramentas necessárias para a compilação e entrega de resultaos do código escrito.

Ao fecharmos esta janela, o software estará apresentado de forma limpa ao programador, etnão iremos na barra de opções que se encontra na parte superior da janela e iremos na opção New, no canto esquerdo. Então ao clicar na opção class uma janela semelhante a anterior será aberta, onde poderá ser configurada as opções do "main" do script e como dito anteriormente, não mexa naquilo que não saiba. Logo, altere apenas o parâmetro Name para declarar um nome ao arquivo.

    Alguns programadores costumam ressaltar sobre a "maldição do programador", uma brincadeira existente neste universo da programação que diz que caso o o programador ao iniciar o seu script, não comece com uma ordem de print na execução com "Hello World", o mesmo terá problemas com o andamento de seu script. Claro que esta é apenas uma brincadeira recorrente e sem fundamento algum, mas se analisar a questão em sí, é uma ideia muito boa. Uma vez que ao nos depararmos com alguma linguagem executável, devemos, por capricho, executarmos um dos comandos mais fáceis inicialmente para vermos se o programa está resondendo sem erro algum, além é claro de ser uma iniciativa para aprender o comando de printagem de texto/string na execução.

    Seguindo então este conceito da maldição do programador, iniciaremos com um comando básico de printagem do Hello World através do Java. Caso já esteja familiarizado com a linguagem C, o comando viria a ser printf(), onde a string se encontraria no meio do comando. No java, a ideia é exatamente a mesma, mas este comando em a ser um pouco mais extenso, devido as particularidades do Java. Sendo então, realizado da seguinte forma:

                System.out.println("Hello World");

    Atente-se ao sensitive case do Java, perceba que a inical do comando é feita por uma letra "S" maiúscula e só será executada caso esteja assim, pois caso esteja no formato minúsculo, o Java entende como um erro e não executará seu comando. Você perceberá que ao tentar executar este script um erro será apresentado ao usuário, um erro de sintaxe pois alguns pré-requisitos serão necessário, particulamente precisaremos do seguinte parâmetro:

                public class main{
               public static void main(String[] args) {
                   System.out.println("Hello World");
               }
           }
  • Variáveis e controle de fluxo

    • Como já é de conhecimento, algo essêncial no processode escrita de um programa, é a utilização de variáveis e o seu gerenciamento, onde uma variável possui a função de armazenar valores que podem ou não ser alterados posteriormente, ou seja, armazenar e trabalhar valores conforme a lógica ditada ao script. Como nas demais linguagens, toda variável possui um tipo para seus dados, o qual uma vez indicado permanecerá imutável, seguindo o conceito de implementação abaixo:

                tipo nomeVariável;

    tendo-a declarado, será possível trabalha-la de forma a assumir e alterar valores imbutidos através do sinal de igualdade para atribuição de valor. Outro método que será muito usado, será o comando de printagem do valor da variável para o usuário, seguindo aquele mesmo conceito já conhecido com o System.out.println() recebendo como parâmetro a variável em questão, como no exemplo abaixo:

                public class main{
                public static void main(String[] args){
                    tipo variável
                    System.out.println("A variável em questão é de valor: "+variável);
                }
            }

    Podemos então estimar que o conceito de variáveis, que já é conhecido, se permanece igual, onde os tipos de variáveis (int, float, boolean, char e etc) permanecem o mesmo para o Java, assim como os seus operadores que trabalham em conjunto com essas. Sendo eles operadores matemáticos (+,-,*./ e etc) e operadores de igualdade (=,>,>= e etc). Porém, algo que vale ressaltar, é que por o Java ser uma linguagem de alto nível, algumas particularidades são presentes nesta linguagem de programação, como por exemplo as variáveis do tipo string. No caso do C e de outras linguagens, devemos declarar uma variável do tipo char para que possamos inserir um caracter nesta variável, o mesmo vale para colocarmos uma série de caracteres (como uma frase, por exemplo), no entanto este deve ser um vetor de caracteres do tipo char. No caso do Java, uma de suas JDK's já nos fornecem o tipo String para as variáveis, podendo assim declarar toda uma cadeia de caracteres do tipo String, como no exemplo abaixo:

                public class main{
                public static void main(String[] args){
                    String variável
                }
            }

    Algo que vale ressaltar também, é o comando para transformaão de variável, ou seja, atribuir que uma variável do tipo float receba o conteúdo de uma variável do tipo int por exemplo. Isso se da pelo simples fato de que variáveis de diferentes tipos, não podem ser trapalhadas de forma igual, ou seja, uma variável do tipo float não pode receber o conteúdo de uma variável do tipo int, pois elas possuem tipos diferentes (assim como qualquer outro tipo de variável, seria como misturar água e óleo). Porém, o comando de transformação ou casting realizaça um comando ao programa que irá transformar os tipos das variáveis, permitindo assim este tipo de trabalho.

    É muito simples, imagine que como no exemplo dito acima, existem duas variáveis: int X e float Y. Logo, se houver um comando como: X = Y, o programa irá crachar e apresentar erro, pos são variáveis de tipos diferentes. Portanto, a variável Y dever´ser transformada para o tipo int da seguinte forma:

                float Y = 5;
            int X = (int) Y;

    Isso fará com que o conteúdo da variável Y seja transformado e armazenado na variável X. Porém, saiba que este processo de transformação segue algumas regras, pois nem todo conteúdo pode ser transformado para um tipo específico (como por exemplo transforma uma variável char para int).
