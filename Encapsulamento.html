Encapsulamento - Orientação a objetos

O segundo pilar entre os importantes da orientação a objetos, sendo a técnica que faz com que detalhes internos e funcionamento dos métodos de uma classe permaneçam ocultos, ou seja, é o processo de segurança dos detalhes e funcionamentos de uma classe, de forma que terceiros não consigam acessar os dados de forma direta, evitando erros e falhas com o funcionamento da classe. Pode-se dizer também, que o processo de encapsulamento tem como função esconder a complexidade do script para o usuário.

Imagine então que um usuário deverá se preocupar apenas com o uso do programa, fazendo com que o programador "esconda" a engenharia/lógica de execução do programa.

Por exemplo, imagine uma classe Carro que contenha métodos como (Ligar, Acelerar, Parar, Desligar) e os atributos (Modelo, Marca, Ano, Km). Neste caso, se o usuário tiver acesso direto aos atributos do carro, com toda certeza iriam ocorrer falhas, fazendo toda a estrutura do carro quebrar, ou seja, tal complexidade deve ficar disponível apenas para o desenvolvedor do carro, pois ele entende e compreende a arquitetura e a lógica da execução.

Diante disso, o processo de emcapsulamento cria métodos que permitam a interação do usuário, com os atributos, de maneira indireta. No exemplo do carro, além dos métodos adotados, teriamos os métodos de encapsulamento, como:

  • Modelo - VerModelo() e AlterarModelo();
  • Marca - VerMarca() e AlterarMarca();

Diante disso, a primeira coisa que devemos fazer, é inserir o comando private nos atributos que foram sensíveis para a classe, ou seja, todos aqueles atributos presentes em uma classe, que são sensíveis de mais (que não podem sofrer interação direta com o usuário ou outras classes) devem receber o comando de modificação de acesso, o private. Isto define que apenas a classe em questão, poderá modificar tal atributo, como por exemplo:

        //Atributos
            private String MarcaCarro;

Tendo um atributo fechado/privado, só poderemos alterar e manipular o valor do atributo através de métodos publicos. No caso do exemplo acima, podemos criar o método InserirMarca(), onde podemos alterar este valor de acordo com o parâmetro que o usuário inserir (Lembre-se de inserir o comando public nos métodos que possuem relação com atributos privados). Para um melhor entendimento, segue o exemplo:

        //Atributos
            private String MarcaCarro;
            
        //Metodos
            public void InserirMara(String marca){
                this.Marca = marca;
            }

Ou seja, a interação direta com o atributo está bloqueada, mas a interação indireta (através de métodos) está permitida para uso e manipulação do mesmo. Aconselha-se também, utilizar de validadores no momento de criação do método, de forma que o programa se torne inteligente suficiente para não permitir valores inválidos, por exemplo:

        //Atributos
            private String MarcaCarro;
            
        //Metodos
            public void InserirMara(String marca){
                if(marca.isEmpty()){
                    this.Marca = marca;
                }else{
                    View.Erro("MARCA INVÁLIDA");
                }
            }

Esses métodos podem, geralemnte, variar entre 2 possíbilidades, onde será possível alterar o valor de um atributo privado ou, poder exibir o valor de um atributo privado. Para isso, nomeamos esses atributos de get (exibir) e set (modificar). Ou ao clicar com o botão direito sobre a tela, podemos ir na opção source e geneate Gtters and Setters.

Outra diferença que deverá ser implementada, é em relação aos construtores que ao invés de receberem o atributo direto com this., deverão receber os Setters. Como no exemplo abaixo:

        //Atributos
            private String Marca;
            
        //Constructor
            Carro(String marca){
                SetMarca(marca);
            }
            
        //Metodos
            public void SetMarca(String marca){
                if(marca.isEmpty()){
                    this.Marca = marca;
                }else{
                    View.Erro("MARCA INVÁLIDA");
                }
            }
            
            public void GetMarca(){
                return this.Marca;
            }

Você perceberá que utilizar o processo de encapsulamento com ArrayList pode ser um pouco mais trabalhoso ao utilizar dos Get/Set. Portanto, é indicado utilizar de métodos públicos que trabalhem e validem o ArrayList em questão (devendo ainda sim, trancá-lo com o comando private). Como no exemplo a seguir:

        //Atributos
            private ArrayList<Carro> carros = new ArrayList<Carro>();
            
            
        //Metodos
            public void inserirCarro(Carro car){
                this.carros.add(car);
            }
            
            public void removerCarro(Carro car){
                this.carros.remove(car);
            }
