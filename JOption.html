Trabalhando com JOption - Orientação a objetos

Tendo aprendido os conceitos básicos e introdutórios sobre o uso de classes e objetos, atribuições de atributos e módulos, vale ressaltar uma importante ferrament utilizada em 90% dos programas e scripts criados. Que é exatamente o processo de abstração e interação de dados do próprio programa com o usuário em questão, ou seja, ferramentas que permitem o usuário interagir diretamente com o progrma, inserir dados e etc. Em C, por exemplo, utilizamos e abusamos muito dos comandos printf() e scanf() que realizam essas interações diretas com o usuário. Porém, com o java e a orientação a objetos, faz-se necessária a chamada de bibliotecas e suas funções, as quais serão passadas a seguir.

Já aprendemos que podemos utilizar de "atalhos" em alguns comandos para com o Java, através de seu prefixo e ctrl + espaço e faremos o mesmo quanto a chamada das funções Java para interação com o usuário. Para isso, inserimos o prefixo joption e em seguida o comando de atalho. Feito isso, será auto encrementado o comando JOptionPane e acima do script, terá o comando import javax.swing.JOptionPane; Que é exatamente a chamada da biblioteca que desejamos usar.

Esse comando de JOptionPane funcionará como o printf() ou scanf(), dependendo da forma como o chamamos. Por exemplo:

  • JOptionPane.showMessageDialog(null, mensagem) - Comando de exibição de mensagens para o usuário.

  • JOptionPane.showImputDialog(mensagem) - Comando de interação do usuário, com a inserção de valor/dado (Utilizamos de uma variável recebendo este comando, para que seja inserido o valor que o usuário escrever, na variável). Como no exemplo a seguir:
    •             String Nome = JOptionPane.showImputDialog(" Insira o seu nome: ");
