Abtsração e Classes - Orientação a objetos

Podemos dizer que a oientação a objetos possui 4 pilares que a definem, de forma a comporem todo o processo de criação de um projeto a partir da orientação a objetos. O primeiro desses pilares que iremos estudar é o de Abstração, que como o seu nume sugere, é a o processo em que o programador irá coletar todo tipo de informção necessária para a criação do seu projeto, ou seja, é a reflexão utilizada para a definição de entidades do mundo real. Na abstração, é possível diferenciar o que é relevante e o que não é para o desenvolvimento do projeto.

Logo, podemos definir que de certa forma, o processo de abstração vem a ser o primeiro e principal pilar para a defnição e criação de um projeto, pois a partir dele podemos definir o esqueleto e darmos iniciativa ao projeto, além de definirmos um norte para o rumo e objetivo que o projeto terá. Algumas pessoas caem na ilusão de que por este pilar de bastração ser apenas o de coleta de informação, é o mais fácil dos processos. Porém este é um erro muito grave, uma vez que definir o que é realmente importante para um projeto, éu pocesso muito delicado e certeiro. Pois imagine que projetos custam dinheiro e se o seu projeto apresentar a falta de algo ou a presença de algo irrelevante, o custo poderá ser maior para sua produção.

Portanto, um programador deve desenvolver um senso de perspectiva e análise de um problema, de forma a abstrair apenas o que for necesspario para o seu projeto. Em orientação a objetos, o principal mecanismo de abstração, são as chamadas Classes, que se comportam como as entidades que irão atuar no projeto. Imagine por exemplo que um programador foi contratado para desenvolver um sistema que irá gerenciar uma clínica veterinária. Logo, a entidade principal deste projeto serão os animais que devem ser analisados pelo doutor certo? Então, as classes viriam a ser os tipos de animais que esta clinica irá tratar, como por exemplo cachorros e gatos.

  • Classes

    • Como dito acima, as classes serão então o campo de definição das entidades que um projeto irá trabalhar, as quais serão compostas por atributos e métodos. Esses atributos e métodos irão ajudar a compor/definir as classes do projeto, ou seja, serão informações/dados que definem as características da classe.

    Se voltarmos ao exemplo da clínica veterinária, digamos que uma classe gato foi definida e agora precisaremos analisar e abstrair as características essênciais e relevantes que definem um gato para um veterinário, como por exemplo se este gato se alimenta bem, sua saúde, situação do pelo, o peso, altura e etc. Perceba que essas ifnromações irão ajudar o doutor de alguma forma, irão cooperar para a sua análise.

    • Atributos

      • Os atributos de uma classe são todas as características que compõem a classe.

      No exemplo da classe gato, podemos definir como atributos a raça, a altura, peso e etc

    • Métodos

      • Os métodos de uma classe são todas as ações/atividades e comportamentos que uma classe pode realizar (é recomendado inserir nomes de métodos como modos verbais = inserir(), apresentar(), sacar()).

      No exemplo da classe gato, podemos definir como métodos se o gato bebe leite, se ele come, o miado e etc.

    Outro termo interessante para o conhecimento nesta parte, é o de instâncias, que são os resultados que uma classe pode gerar, ou seja, os objetos que uma classe irá definir. Em um segundo exemplo, podemos definir então que a classe seria como uma forma de bolo, o molde que dá formato ao bolo. Logo, todo bolo criado terá exatamanete o mesmo molde, porém diferentes bolos podem ser criados, como chocolate, cenoura e etc. Portanto, a perspectiva é a mesma, pois vários objetos/instâncias (bolos) podem ser gerados a partir de uma mesma classe (molde) e independete do sabor do bolo, o seu formato será o mesmo.

    Um terceiro exemplo que podemos dar, este mais voltado ao universo de programação, é que a criação de uma classe está relacionada de forma semelhante a definicção de structs, onde uma struct criada (classe) terá N características que a definem (atributos e métodos).

Tendo estes conceitos compreendidos, partiremos então para uma situação mais frequente do processo de abstração e criação de classes, e como iniciarmos este procedimento. Uma ótima dica que pode ajudar muito neste processo, é ter uma planilha em que podemos definir certinho as classes do projeto e os atributos e métodos de cada classe.

Imagine então que iremos criar um aplicativo de celular extamaente igual ao famoso Tinder, uma "rede social" destinada a promover encontros e relacionamentos entre as pessoas. Então, devemos partir do primeiro pilar da orientação a objetos, o processo de abstração. Ou seja, definirmos as classes e composições destas classes de forma relevante. Temos então, de maneira rápida, as seguintes definições:

Perceba então que essas são algumas das classes que irão regir o projeto (o aplicativo Tinder nesse caso) e após conseguirmos abstrair e registrar tais informaçõs, partiremos para a implementação das classes no Java.

  • Classes no Java

    • O processo de criação/definição das classes no Java, já é parcialmente conhecido, uma vez que já aplicamos a abertura do script. Logo, na janela de canto esquerdo (configurações do projeto) podemos com o botão direito, apertar em default package ou src e selecionarmos a opção New e Class. Implemente o nome que a classe em questão receberá e estará pronta para ser implementada.

    A partir de então, podemos definir os atributos e métodos da classe como se fossem variáveis a serem usadas, como no exemplo abaixo:

                public class Usuario {
                 //Atributos
                 String foto,nome,desc;
                 int idade;
                 
                 //Métodos
                 void DarMatch() {
                    System.out.println(this.nome+" Deu match...");
                 }
                 void PostarFoto() {
                    System.out.println("Foto postada...");
                 }
                 void EnviarMsg() {
                    System.out.println("Mensagem enviada...");
                 }
                 void Denunciar() {
                    System.out.println(this.nome+"Denunciou...");
                 }
            }

    Perceba então, que quanto a definição dos atributos, é exatamente como declarar variáveis em um script. Já quanto aos métodos, devemos defini-los como funções que irão retornar algum resultado. Perceba também o uso do comando this. na chamada dos métodos. Este faz com que os atributos definidos sejam apresentados durante o processo de printagem e a necessidade do this. é para que especificamente os valores desta classe no momento, sejam apresentados.

    Em relação aos métodos, entenda que os atributos podem ser trabalhados de maneira comum entre eles, como por exemplo incrementar ou diminuir o valor de algum atributo, atribuir novos valores e etc, ou seja, fica a mercer do programador utilizar desses métodos de acordo com sua vntade/necessidade.

    Agora, devemos chamar esta classe para o main/executor de forma que script principal use esta classe definida. Para isso, iremos entender uma classe como se fosse um tipo de variável criada, ou seja, neste caso, foi criada a "variável" (classe) Usuario:

                public class Exec{
                public static void main(String[] args){
                    Usuario Marcos = new Usuario();
                }
            }

    Atente-se ao parâmetro new Classe(), pois este se faz necessário sempre que um objeto for criado para tal classe. Tendo definido tanto a classe como o objeto/entidade para o projeto, pdemos atribuir então, valores para este objeto criado (seguindo a mesma lógica usada nas implementações de structs):

                public class Exec{
                public static void main(String[] args){
                    Usuario Marcos = new Usuario();
                        Marcos.nome = "Marcos Jr";
                        Marcos.idade = 21;
                        Marcos.desc = "Sou pika...";
                        
                        Marcos.DarMatch();
                }
            }

    Dessa forma, o objeto Marcos da classe Usuario foi definido e usado pela main principal. E como dito, perceba que sobre os atributos, sua definição de valores é exatemnte igual a variáveis de uma struct e quanto aos métodos, é exatamente igual a chamar uma função. Vale ressaltar, que os atributos que tiverem valores atribuidos no momento de criação da classe, possuirão valores default para qualquer novo objeto criado, ou seja, no caso do exemplo acima, se inserirmos o valor 21 para a idade no momento de criação da classe, todos os novos objetos criados a partir dela na main irão obter por default o valor 21 como idade. É claro que se for desejo do usuário alterar estes valores default durante a main, também será possível.

    Traremos agora um outro exemplo, para que essa ideia da criação da classe com seus parâmetros e seus objetos, seja fixada. No exemplo a seguir, criaremos um programa baseado uma conta bancária, onde a classe deve obter e trabalhar os dados de um cliente:

                public class Conta{
                //Atributos
                    String Nome;
                    int Numero;
                    int Agencia = 222;
                    float Saldo = 0;
                    
                //Métodos
                    void sacar(float valor){
                        this.saldo = (this.saldo - valor);
                        System.out.println("\n [CONTA:] Saque realizado com sucesso! \n");
                    }
                    
                    void depositar(float valor){
                        this.saldo = (this.saldo + valor);
                        System.out.println("\n [CONTA:] Deposito realizado com sucesso! \n");
                    }
                    
                    void transferir(Conta destino, float valor){
                        this.sacar(valor);
                        destino.saldo = (destino.saldo + valor);
                    }
                    
            
            }

    Neste caso, temos atributos com valores definidos por default (Agencia e Saldo), logo todo objeto criado a partirdesta classe, terá estes valores inicialmente, para tais atributos. Já no método de sacar(), perceba que foi exigido uma variável valor como parâmetro (logo, o cliente precisa passar o valor que deseja sacar) e ao executar tal método, o valor que foi sacado será descontado de seu saldo atual.

    No caso do método de transferir(), perceba que os parâmetros a serem usados são os objetos da classe Conta (destino = Conta que está recebendo) e na sua implementação, esta chamou outro método para trabalho. Seguindo a exata mesma lógica de usar uma função dentro de outra função em C, certo? No java com orientação a objetos, a ideia é exatamente a mesma, estamos apenas chamando um método dentro de outro método.

    Já na main() podemos ter exemplificado da seguinte forma:

                public class Exec{
                public static void main(String[] args){
                    Conta c1 = new Conta();
                        c1.Nome = " Marcos Jr ";
                        c1.depositar(500);
                        System.out.println("\n O saldo do(a) "+c1.Nome+" é de: "+c1.Saldo);
                        
                    Conta c2 = new Conta();
                        c2.Nome = " João ";
                        c2.depositar(200);
                        System.out.println("\n O saldo do(a) "+c2.Nome+" é de: "+c2.Saldo);
                        
                    c1.transferir(c2, 500); 
                    System.out.println("\n O saldo do(a) "+c1.Nome+" é de: "+c1.Saldo);
                    System.out.println("\n O saldo do(a) "+c2.Nome+" é de: "+c2.Saldo);
                        
                }
            }

    Aprederemos agora sobre uma variável universal, a qual é possível apresentarmos todos os valores dos atributos de um determinado objeto. Basicamente, esta é uma chamada de memória em que podemos exibir diretamente o que está alocado na memória relacionada a determinado objeto/classe. Primeiramente, tente executar o comando de printagem do objeto: System.out.println(objeto) e perceba o valor a ser exibido. É certo que será exibido um valor como: classe@algum número. Este é o exato endereço de memória do objeto, ou melhor, uma referência ao endereço do objeto. Isso se da pelo fato de que a orientação a objetos trabalha com reference type (referência a endereços de memória e seus valores).

    Ou seja, com a orientação a objetos, trabalhamos com referências aos endereços de memória em que os objetos/classes estão alocados e ao printarmos um objeto, temos a exibição direta de seu endereço referenciado. Porém, é possível alterar este parâmetro, de forma que o que seja exibido, sejam as informações da classe. Para isso, usamos do comando:

                public String toString(){
            
            }

    Este campo trabalha os dados de exibição de um endereço de memória da classe. Dessa forma, podemos alterá-lo para exibirmos o que quisermos, como por exemplo a apresentação de todos os dados da classe em questão:

                public String toString(){
                return "[CONTA: ] - USUARIO: " + this.Nome + "\n[CONTA: ] - SALDO: " + this.Saldo;
            }
  • Construtores

    • Algo que serve de auxílio no processo de criação de uma classe, são os chamados construtores, que funcionam como um parâmetro inicial obrigatório na declaração de algum objeto para tal classe, ou seja, ao defirnirmos construtores para uma classe, todo objeto que criado a partir dela, deverá obrigatoriamente passar os parâmetros exigidos. Esses parâmetros são variáveis iniciais que irão definir e construi o objeto (os quais são definidos pelo programador).

    Imagine pelo exemplo acima,que para a criação de uma conta de usuário para o banco, é necessário definir logo de cara o número da conta e o dono da mesma. Usaremos então de contrutores para exigirmos isso para os objetos da classe conta, como a seguir:

                public class Conta{
                //Atributos
                    int Numero, Agencia = 934;
                    double Saldo;
                    String Dono;
                    
                //Construtor
                    Conta(int numero, String dono){
                        this.Numero = numero;
                        this.Dono = dono;
                    
                    }
            }

    Perceba então que os parâmetros numero e dono foram exigidos e depois atribuidos aos atributos da classe através do this.. Dessa forma, quando criado um objeto da classe conta, será necessário passar inicialmente esses 2 parâmetros. Além é claro de que o script se torna muito mais legível e curto, evitando declarações de variáveis que já podem ser passadas pelo construtor. Como no exemplo a seguir:

                public class Exec{
                public static void main(String[] args){
                    Conta c1 = new Conta(001,"Marcos Jr");
                        
                    Conta c2 = new Conta(002,"João");

                        
                }
            }

    Vale ressaltar que o processo de abtsração é muito exigente na criação do construtor, pois recomenda-se passar no contrutor apenas aqueles atributos que são realmente necessários passar inicialmente. Como no caso do atributo saldo, pois uma conta pode muito bem ter um saldo zerado. Logo, não é necessário passar este parâmetro. Já no caso dos atributos numero e dono, são extremamente necessários para a criação de uma conta certo?
