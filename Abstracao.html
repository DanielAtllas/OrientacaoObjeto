Abtsração e Classes - Orientação a objetos

Podemos dizer que a oientação a objetos possui 4 pilares que a definem, de forma a comporem todo o processo de criação de um projeto a partir da orientação a objetos. O primeiro desses pilares que iremos estudar é o de Abstração, que como o seu nume sugere, é a o processo em que o programador irá coletar todo tipo de informção necessária para a criação do seu projeto, ou seja, é a reflexão utilizada para a definição de entidades do mundo real. Na abstração, é possível diferenciar o que é relevante e o que não é para o desenvolvimento do projeto.

Logo, podemos definir que de certa forma, o processo de abstração vem a ser o primeiro e principal pilar para a defnição e criação de um projeto, pois a partir dele podemos definir o esqueleto e darmos iniciativa ao projeto, além de definirmos um norte para o rumo e objetivo que o projeto terá. Algumas pessoas caem na ilusão de que por este pilar de bastração ser apenas o de coleta de informação, é o mais fácil dos processos. Porém este é um erro muito grave, uma vez que definir o que é realmente importante para um projeto, éu pocesso muito delicado e certeiro. Pois imagine que projetos custam dinheiro e se o seu projeto apresentar a falta de algo ou a presença de algo irrelevante, o custo poderá ser maior para sua produção.

Portanto, um programador deve desenvolver um senso de perspectiva e análise de um problema, de forma a abstrair apenas o que for necesspario para o seu projeto. Em orientação a objetos, o principal mecanismo de abstração, são as chamadas Classes, que se comportam como as entidades que irão atuar no projeto. Imagine por exemplo que um programador foi contratado para desenvolver um sistema que irá gerenciar uma clínica veterinária. Logo, a entidade principal deste projeto serão os animais que devem ser analisados pelo doutor certo? Então, as classes viriam a ser os tipos de animais que esta clinica irá tratar, como por exemplo cachorros e gatos.

  • Classes

    • Como dito acima, as classes serão então o campo de definição das entidades que um projeto irá trabalhar, as quais serão compostas por atributos e métodos. Esses atributos e métodos irão ajudar a compor/definir as classes do projeto, ou seja, serão informações/dados que definem as características da classe.

    Se voltarmos ao exemplo da clínica veterinária, digamos que uma classe gato foi definida e agora precisaremos analisar e abstrair as características essênciais e relevantes que definem um gato para um veterinário, como por exemplo se este gato se alimenta bem, sua saúde, situação do pelo, o peso, altura e etc. Perceba que essas ifnromações irão ajudar o doutor de alguma forma, irão cooperar para a sua análise.

    • Atributos

      • Os atributos de uma classe são todas as características que compõem a classe.

      No exemplo da classe gato, podemos definir como atributos a raça, a altura, peso e etc

    • Métodos

      • Os métodos de uma classe são todas as ações/atividades e comportamentos que uma classe pode realizar.

      No exemplo da classe gato, podemos definir como métodos se o gato bebe leite, se ele come, o miado e etc.

    Outro termo interessante para o conhecimento nesta parte, é o de instâncias, que são os resultados que uma classe pode gerar, ou seja, os objetos que uma classe irá definir. Em um segundo exemplo, podemos definir então que a classe seria como uma forma de bolo, o molde que dá formato ao bolo. Logo, todo bolo criado terá exatamanete o mesmo molde, porém diferentes bolos podem ser criados, como chocolate, cenoura e etc. Portanto, a perspectiva é a mesma, pois vários objetos/instâncias (bolos) podem ser gerados a partir de uma mesma classe (molde) e independete do sabor do bolo, o seu formato será o mesmo.

    Um terceiro exemplo que podemos dar, este mais voltado ao universo de programação, é que a criação de uma classe está relacionada de forma semelhante a definicção de structs, onde uma struct criada (classe) terá N características que a definem (atributos e métodos).

Tendo estes conceitos compreendidos, partiremos então para uma situação mais frequente do processo de abstração e criação de classes, e como iniciarmos este procedimento. Uma ótima dica que pode ajudar muito neste processo, é ter uma planilha em que podemos definir certinho as classes do projeto e os atributos e métodos de cada classe.

Imagine então que iremos criar um aplicativo de celular extamaente igual ao famoso Tinder, uma "rede social" destinada a promover encontros e relacionamentos entre as pessoas. Então, devemos partir do primeiro pilar da orientação a objetos, o processo de abstração. Ou seja, definirmos as classes e composições destas classes de forma relevante. Temos então, de maneira rápida, as seguintes definições:

  • Classes:

    • Usuário:
      • Atributos:
        • Foto
        • Nome
        • Idade
        • Descrição

      • Métodos:
        • Dar Match
        • Postar foto
        • Enviar Mensagem
        • Denunciar

    • Chat:
      • Atributos:
        • Usuário 1
        • Usuário 2
        • Data
        • Mensagens

      • Métodos:
        • Curtir Mensagem
        • Apagar Mensagem
        • Enviar Mensagem

    • Mensagem:
      • Atributos:
        • Data
        • Texto
        • Visualização
        • Autor

Perceba então que essas são algumas das classes que irão regir o projeto (o aplicativo Tinder nesse caso) e após conseguirmos abstrair e registrar tais informaçõs, partiremos para a implementação das classes no Java.

  • Classes no Java

    • O processo de criação/definição das classes no Java, já é parcialmente conhecido, uma vez que já aplicamos a abertura do script. Logo, na janela de canto esquerdo (configurações do projeto) podemos com o botão direito, apertar em default package ou src e selecionarmos a opção New e Class. Implemente o nome que a classe em questão receberá e estará pronta para ser implementada.

    A partir de então, podemos definir os atributos e métodos da classe como se fossem variáveis a serem usadas, como no exemplo abaixo:

                public class Usuario {
                 //Atributos
                 String foto,nome,desc;
                 int idade;
                 
                 //Métodos
                 void DarMatch() {
                    System.out.println("Deu match...");
                 }
                 void PostarFoto() {
                    System.out.println("Foto postada...");
                 }
                 void EnviarMsg() {
                    System.out.println("Mensagem enviada...");
                 }
                 void Denunciar() {
                    System.out.println("Denunciou...");
                 }
            }

    Perceba então, que quanto a definição dos atributos, é exatamente como declarar variáveis em um script. Já quanto aos métodos, devemos defini-los como funções que irão retornar algum resultado.

    Agora, devemos chamar esta classe para o main/executor de forma que script principal use esta classe definida. Para isso, iremos entender uma classe como se fosse um tipo de variável criada, ou seja, neste caso, foi criada a "variável" (classe) Usuario:

                public class Exec{
                public static void main(String[] args){
                    Usuario Marcos = new Usuario();
                }
            }

    Atente-se ao parâmetro new Classe(), pois este se faz necessário sempre que um objeto for criado para tal classe. Tendo definido tanto a classe como o objeto/entidade para o projeto, pdemos atribuir então, valores para este objeto criado (seguindo a mesma lógica usada nas implementações de structs):

                public class Exec{
                public static void main(String[] args){
                    Usuario Marcos = new Usuario();
                        Marcos.nome = "Marcos Jr";
                        Marcos.idade = 21;
                        Marcos.desc = "Sou pika...";
                        
                        Marcos.DarMatch();
                }
            }

    Dessa forma, o objeto Marcos da classe Usuario foi definido e usado pela main principal. E como dito, perceba que sobre os atributos, sua definição de valores é exatemnte igual a variáveis de uma struct e quanto aos métodos, é exatamente igual a chamar uma função.
