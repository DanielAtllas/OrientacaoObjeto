Herança - Orientação a objetos

O terceiro principal pilae de orientação a objetos se baseia na capacidade que as classes herdam os atributos/métodos de uma classe "universal", chamada de superclasse. Ou seja, a herança permite que classes compartilhem os seus atributos/métodos, através de uma relação hierarquica de "pai-filho". Ela é usada na intenção de reaproveitar código ou comportamentos.

Diante disso, temos tipos de classes para tal contexto, como:

  • Superclasse - Classe universal destributiva, também tida como "classe pai";

  • Subclasses - São as classes originadas da anterior, também tidas como "classes filhas".

Dessa forma, a supe classe irá possuir os atributos/métodos que serão reaproveitados pelas subclasses. Imagine por exemplo 3 classes {Veiculo, Carro, Moto}. Podemos observar que todo carro é um veículo, mas nem todo carro é uma moto (o mesmo para a classe moto). Logo, a classe veiculo é uma superclasse e as demais, subclasses.

Imagine agora que todo veículo possui os atributos {Marca, Ano, Modelo}. Para não repetir os mesmos atributos para a classe moto e carro, apenas a superclasse Veiculo conterá esses atributos universais.

Implementação

Partiremos agora para a implementação deste pilar da herança no script do programa, criamos primeiro então a superclasse que terá os dados universais:

            abstract public class Veiculo{
                //Attributes
                    String Marca, Modelo;
                    int Ano;
            }

Perceba o uso do comando abstract no escopo da classe. Este deve ser inserido pelo progromador e possui a função de transformar a classe em universal, de forma que ela não possa criar objetos puros e diretos.

O segundo passo, é criar a subclasse que herdará os dados da anterior:

            public class Carro extends Veiculo{
                //Attributes
                    int QtdPortas;
            }

Perceba o uso do comando extends superclasse, o qual indica que esta é uma subclasse de outra indicada. Assim, ele possui acesso aos atributos de sua superclasse, mas ninguém acessará seus atributos únicos (QtdPortas):

            public class Exec{
                //Objects
                    Carro uno = new Carro();
                    
                    uno.Marca = "Fiat";
                    uno.Ano = 2018;
                    uno.QtdPortas = 4;
            }

Vale lembrar que uma subclasse, pode se tornar uma superclasse para outras N classes, dependendo do desejo do programador. Para isso, basta repetir os mesmos procedimentos.

Sobrescrita

Com a sobrescrita, conseguimos especializar os métodos herdados das superclasses, alterando o seu comportamento nas subclasses por um mais específico. Esta capacidade, está relacionada aos métodos compartilhados pela superclasse, ou seja, podemos trabalhar/alterar/utilizar os métodos da superclasse de forma indireta, ao serem chamados pelas subclasses. Como no exemplo a seguir:

            abstract public class Veiculo{
                //Methods
                    public void Acelerar(int velocidade){
                        View.Exibirmsg("Acelerando a "+velocidade+"Km/h");
                    }
            }
            
            ==============================================
            
            public class Carro extends Veiculo{
                //methods
                    @Override 
                    public void Acelerar(int velocidade){
                        super.Acelerar(velocidade);
                        View.Exibirmsg("Correndo...");
                    }
            }
